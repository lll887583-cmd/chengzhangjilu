# AGENTS.md

## 项目概况

- 本项目是“成长记录”静态网页 Demo，主目录为 `成长记录/`。
- 入口文件：`成长记录/index.html`、`成长记录/styles.css`、`成长记录/app.js`。
- 当前代码已按静态模块拆分：`pets.js` 管宠物 IP，`data.js` 管积分/奖励/默认配置，`store.js` 管本地状态，`views/` 管页面渲染，`app.js` 管交互。
- 当前没有构建系统和包管理配置，优先按静态网页方式维护。
- 本地预览命令：`./serve.sh` 或 `python3 -m http.server 5173`。
- localhost 访问地址：`http://localhost:5173/成长记录/`。
- 默认入口是“获取积分 - 获得”；点击 Logo 也返回这个默认页。
- 总积分按钮会进入“我的 - 积分记录”详情页。

## 工作原则

- 优先只改用户明确要求的文件和区域，不主动重构无关代码。
- 修改前先快速定位相关文件；用户已给出精确路径时，不做全项目扫描。
- 保留现有中文文案、交互逻辑和视觉风格，除非用户明确要求调整。
- 不覆盖用户未提交或未说明的改动；发现异常变更时先停止并询问。
- 默认不引入构建工具、包管理器、框架或外部依赖；除非用户明确要求。
- 优先保持静态网页可直接通过 `python3 -m http.server 5173` 预览。
- 预览服务必须保持前台或持续会话运行；不要用会被任务结束自动回收的临时后台服务误报“已启动”。

## 快速修改地图

- `成长记录/pets.js`：10 个宠物 IP、属性、领养/复活积分和图片路径。
- `成长记录/data.js`：积分规则、扣减规则、兑换奖励、抽奖奖项、默认初始数据，并 re-export `PETS`。
- `成长记录/store.js`：localStorage 读写、状态补齐、积分扣减、成长记录追加。
- `成长记录/views.js`：视图模块统一出口，通常不直接写页面结构。
- `成长记录/views/shared.js`：页面共享渲染工具，如切换栏、状态卡、积分记录标题。
- `成长记录/views/points.js`：获取/扣减积分页面。
- `成长记录/views/shop.js`：兑换商城/积分抽奖页面。
- `成长记录/views/pet.js`：云宠物/宠物馆页面。
- `成长记录/views/my.js`：我的、成长档案、我的兑换、积分记录页面。
- `成长记录/app.js`：按钮事件、积分/宠物/兑换/核销/抽奖交互逻辑。
- `成长记录/styles.css`：样式入口，只维护 `@import` 顺序。
- `成长记录/styles/base.css`：主题变量、基础元素和字体。
- `成长记录/styles/layout.css`：应用壳、顶部栏、切换栏和基础布局。
- `成长记录/styles/components.css`：通用卡片、按钮、统计卡等复用组件。
- `成长记录/styles/pet.css`：云宠物、宠物馆和宠物详情样式。
- `成长记录/styles/screens.css`：获取积分、兑换商城、我的页面样式。
- `成长记录/styles/navigation.css`：底部导航。
- `成长记录/styles/overlays.css`：Toast、弹窗、核销、宠物详情弹层。
- `成长记录/styles/responsive.css`：窄屏适配。
- `成长记录/icons.js`：内联 SVG 图标集合。
- `成长记录/assets/`：Logo、宠物图片和奖励图片等静态素材。
- `成长记录/assets/pets/`：当前 10 个宠物使用 `.png` 图片，未引用旧素材已清理。
- `成长记录/assets/rewards/`：抽奖等奖励图片。
- `成长记录/docs/`：产品、UI/UX 或接入说明类文档。

## 常见需求入口

- 新增或调整积分任务：优先只改 `成长记录/data.js` 的 `POINT_RULES`。
- 新增或调整扣减任务：优先只改 `成长记录/data.js` 的 `DEDUCT_RULES`。
- 新增或调整兑换商品：优先只改 `成长记录/data.js` 的 `REWARDS`。
- 新增或调整抽奖：优先只改 `成长记录/data.js` 的 `LOTTERY`。
- 新增宠物：先改 `成长记录/pets.js` 的 `PETS`，再放入对应图片到 `成长记录/assets/pets/`。
- 调整默认进入页面、Logo 跳转、总积分跳转：优先改 `成长记录/data.js` 的 `defaultState` 和 `成长记录/app.js` 的 `goHome` / `openRecordsDetail`。
- 调整页面文案：优先改 `成长记录/views/` 中对应页面模块。
- 调整颜色、间距、按钮、卡片：优先改 `成长记录/styles/` 中对应模块；通用按钮/卡片改 `styles/components.css`。
- 调整积分、兑换、抽奖、宠物操作流程：优先改 `成长记录/app.js`，必要时同步 `store.js`。

## 当前业务配置

- 获得积分任务在 `POINT_RULES`，当前为：画画、粘土手工、11点入睡、学习汉字、学习数学、完成学校作业、打卡英语、练字。
- 扣减积分任务在 `DEDUCT_RULES`，当前为：不好好吃饭、晚睡、无理取闹、骂人、丢东西、打人。
- 兑换奖励在 `REWARDS`，当前包含：买 2 个零食、买 1 个玩具、吃肯德基、看 40 分钟电视、玩 30 分钟游戏。
- 宠物馆当前是 10 个 IP，桌面端按 5 列 x 2 排展示，窄屏按 2 列展示。
- 宠物名字不要带 emoji；镇馆之宝标记由 `featured: true` 控制。
- 云宠物主卡片不展示宠物介绍、攻击、防御、速度；这些只在宠物详情弹窗中展示。
- 宠物详情弹窗底部无按钮，点击右上角关闭或点击遮罩空白处关闭。

## 速度优先约定

- 用户要求“速答”“快点”“只记录”“只追加”时，按最小可行改动执行。
- 减少验证范围：如果用户明确说“无需跑测试”“无需全局检查”，不要运行测试、构建或大范围扫描。
- 对简单文案、记录类修改，只检查目标文件是否写入成功即可。
- 只有在改动影响运行逻辑、样式结构或用户要求验证时，才进行本地预览、浏览器检查或额外测试。

## 成长记录类任务

- 追加成长记录或默认记录时，优先沿用现有格式，不改历史记录。
- 若用户没有要求整理，不进行分类、重排、摘要或格式迁移。
- 若需要新增模板，保持字段简短、清晰、适合持续追加。
- 数据目前暂存在浏览器 localStorage；不要把用户本地数据假定为已同步到云端。

## 验证约定

- 只改文档、注释或配置文案时，检查目标文件写入即可。
- 改 JS 交互逻辑时，至少做静态语法/关键路径检查；必要时启动本地预览。
- 因项目使用 ES module 且无 `package.json`，本地 JS 语法检查可临时复制为 `.mjs` 后执行 `node --check`，不要直接对 `.js` 使用 `node --check`。
- 改 CSS 结构或关键布局时，优先用本地预览地址做浏览器视觉检查。
- 每次完成可视化或交互改动后，回复中都附上 localhost 预览地址：`http://localhost:5173/成长记录/`。
- 用户明确说无需验证时，按最小检查执行。
- 如果用户反馈 `ERR_CONNECTION_REFUSED`，先检查 `lsof -nP -iTCP:5173 -sTCP:LISTEN`，再启动持续运行的 `python3 -m http.server 5173`。
